AnchorNormalization gadget
Input Formats:
  1. CVideoFrame
  2. CVideoFrame (WhiteRefInput)
  3. CVideoFrame (BlackRefInput)
Outputs Formats:CVideoFrame
Duplex Formats: NONE
Location in gadget tree: Filters\Video\color&brightness\ for grouped by type or Video\color&brightness\ by default

AnchorNormalization gadget makes normalization by comparing with samples. Before start gadget it's required to load one or two reference images to pin 2 or 3 of the gadget (White Reference and/or Black Reference).

If white and black images were not received, input image will be sent to the output pin without changes.

If only the white video data frame was received, every pixel of video data frame received in the first input pin will be normalized according to correspondent pixel in white image:

Brout = BrIn • Max / Brwhite

If only the black video data frame was received, every pixel of video data frame received in the first input pin will be decreased for correspondent pixel in black image:

Brout = BrIn - Brblack

If both images (black and white) were received, normalization will be done as follows:

Brout = (BrIn - Brblack) • Max / (Brwhite - Brblack)

The converted video data frame is streamed to the output pin.

 

Gadget has no setup dialog.
