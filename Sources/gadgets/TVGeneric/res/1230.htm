Calculator gadget
Input Formats:CQuantityFrame, or CBooleanFrame, or CTextFrame
Outputs Formats:CQuantityFrame
Duplex Formats: NONE
Location in gadget tree: Filters\Generic\ for grouped by type or Generic\ by default

Calculator is a gadget for performing mathematical operations parameterized by input data.

Each time calculator receives all necessary data to its inputs, it performs the operation which is described by its setup parameter “Expression”.

Expression is a string in a common mathematical format, like

1 + 2
3.25 * (.5 – 2.) / (-1 + 6 ^ 2)

Expressions support operations +, -, *, /, ^ (power), and also ( ) for sub-expressions.

Expressions can contain parameters, i.e. values obtained from inputs, in the form $n, where n is the number of input pin from which the parameter should be read (pins are enumerated top-down starting with 1). n can be either a constant or a valid sub-expression. For instance:

1 + $1
$1 / $2
$(2 + (-1) ^ $1)

Parameters can be provided by data packets of 3 types: quantity (preferred) – double or integer, text (interpreted as common ASCII representation of double or integer value, exponential representation not supported), Boolean (true = 1, false = 0).

Expressions can also contain functions. Currently supported are exp(X), log(X) where X is a constant or any valid expression.

Calculator returns result in a quantity packet. All calculations are performed with floating-point values by default. Number of parameters can be varied by changing the number of gadget input pins in setup (parameter “Vars”). Currently up to 9 parameters per Calculator are supported.

Note that due to a bug in parent gadget Calculator does not perform any operation if it has only 1 pin. A temporary overcome is creating 2 pins and sending identical packets to both of them, using only one as actual parameter. .

 

Gadget setup dialog has following parameters:

Vars - number input variables, or input pins

Expression - expression to be calculated.
