Buffer gadget

Input Formats:

Any type of the data

Outputs Formats:

same as input data type

Duplex Formats:

CQuantityFrame or any

Location in gadget tree:

Filters\Generic\ for grouped by type or Generic\ by default

Buffer gadget stores up to 500 data packets in circle buffer to enable controlled examing of them. The data packets received in input pin are stored in the buffer. The original data packets are sent \ not sent to output pin according to the value of ‘Transparent’ parameter. If the buffer is full (i.e. the number of incoming data packets exceeds the value of ‘BufferSize’), the earlest data packet in the buffer will be deleted and a new one is inserted instead . If quantity data packet with the value ‘1’ is received in control pin, the previous data packet is streamed to output pin.

The control of access to data in the buffer is implenented through Control pin.  At receiving CQuantityFrame the gadget push out the data packet with number which it contain. Negative values interpreted as nil. Any other packets in the Control Pin just push out data packet with 0 offset.

 

Gadget setup dialog has following parameters:

Transparent - defines weather the data packets received in input pin will be sent (after a copy was stored in buffer) to output pin

BufferSize - sets the number of data packets that will be stored in the buffer. Possible values: 1-500.

PureFIFO – if false then gadget works as described above. If true then gadget works as usual FIFO without possibility for data frame repeating, stepping back and so on. Control pin in this case works as follow:

When text frame with label “BufControl” received, if content is “Reset” then all data inside buffer is cleared and text frame with content “Buffer Cleared” and label “BuffOK” will be sent to control pin.

When text frame with label “BufControl” received, if content is “Next” then next frame from buffer will be sent to output pin. If buffer is empty, i.e. all data sent, the text frame with content “Buffer Empty” and label “BuffError” will be sent to output pin.

When text frame with label “BufControl” received, if content is “GetBufStatus” then text frame “BuffSize=<max frame number>;Filled=<# of available frames>;” will be sent to output pin. If buffer is empty (all data sent) text frame with content “BuffSize=<max frame number>;Filled=0;” and label “BuffStatus” will be sent to output pin.

Any Boolean frame or Quantity frame will lead to sending of next frame from buffer to output pin. If buffer is empty, then text frame with content “Buffer Empty” and label “BuffError” will be sent to output pin.
