avt3_11 gadget
 
   Input Formats:      transparent, used for software triggering

Outputs Formats:  CVideoFrame

Duplex Formats:    text

Location in gadget tree: Captures\Video\capture for grouped by type or Video\Capture by default

Streaming video captured in AVT (Allied Vision Technologies) camera used with AVT 2.X SDK driver siftware.

Data processing - Video frames can be streamed via AVT camera in the following methods:

  • No trigger - in which the camera grabs frames according to the predetermined frame rate set in dialog box. In this method, Trigger parameter must be set to ‘off’.
  • Hardware trigger - in which external hardware (controller) is connected to the camera and triggers the grabbing of frames. In this method, Trigger parameter must be set to the appropriate value (Tr1 \ Tr1Inv \ etc.) to enable the receiving of the DoGrab signal.
  • Software trigger - in which the camera grabs one frame per data packet received in input pin. In this method, Trigger parameter must be set to any value different from ‘off’ (Tr1 \ Tr1Inv \ etc.) and not receive trigger signals from external hardware.

A complete description of current setup dialog parameters is printed to a text data packet and sent to control pin as a response to text data packet containing the command ‘list’ received in control pin. The value of a specific setup dialog parameter is printed to a text data packet and sent to control pin as a response to text data packet containing the command ‘get’ and the parameter’s name received in control pin:

  • get Format - invokes the sending of ‘Format’ parameter value to control pin. Possible values: 0 - 10 where 0 = Y8, 1 = YUV411, 2 = YUV422, 3 = YUV444, 4 = RGB8, 5 = Y16, 6 = RGB16, 7 = SY16, 8 = SRGB16, 9 = RAW8, 10 = RAW16.
  • get FrameRate - invokes the sending of ‘FrameRate’ parameter value to control pin. Possible values: 1 - 5 where 1 = 3.75fps, 2 = 7.5fps, 3 = 15fps, 4 = 30fps, 5 = 60fps.
  • get Resolution - invokes the sending of ‘Resolution’ parameter value to control pin. Possible values: 0 - 7 where 0 = 160 x 120, 1 = 320 x 240 2 = 640 x 480, 3 = 800 x 600, 4 = 1024 x 768, 5 = 10280 x 960, 6 = 1600 x 1200, 7 = Scalable.
  • get Brightness - invokes the sending of ‘Brightness’ parameter value to control pin. Possible values: 0 - 255.
  • get Gamma - invokes the sending of ‘Gamma’ parameter value to control pin. Possible values: 0 \ 1.
  • get Shutter - invokes the sending of ‘Shutter’ parameter value to control pin. Possible values: 1 - 4095.
  • get Gain - invokes the sending of ‘Gain’ parameter value to control pin. Possible values: 0 - 680.
  • get ROI - invokes the sending of ‘ROI’ parameter value to control pin. Possible values: Xleft: 0 - image width, Ytop: 0 - image height, Xright: Xleft - image width, Ybottom: Ytop - image height. Note that ROI characteristic appears only if format is set to ‘Scalable’ and that it is relative to the full frame (for example, if ROI is set to (10, 10, 30, 30) and then set again to (10, 10, 30, 30), the same portion of the frame will be shown).
  • get Trigger - invokes the sending of ‘Trigger’ parameter value to control pin. Possible values: 0 - 5 where 0 = no trigger, 1 = Tr1, 2 = Tr1Inv, 3 = Tr2, 4 = Tr2Inv.
  • get Shutt_us - invokes the sending of ‘Shutt_us’ parameter value to control pin. Possible values: 1 - 80000.
  • get FrameInfo - invokes the sending of ‘FrameInfo’ parameter value to control pin. Possible values: 0 = no additional information is sent with the frame / 1 = additional information is sent with the frame.
  • get TrigDelay_us - invokes the sending of ‘Shutt_us’ parameter value to control pin. Possible values: 1 – 80000 microseconds.
  • get Grab - invokes the sending of ‘Grab’ parameter value to control pin.
  • The value of a specific setup dialog parameter can be altered as a response to text data packet containing the command ‘set’, the parameter’s name and its value received in control pin, for example: set Resolution(7) will set the Resolution characteristic to scalable. The list of parameters and their possible values is given above.

Gadget Setup has following option:

Select Camera Setup Dialog

avt3_11 Setup Dialog

  • Camera - sets the AVT camera from which the video stream is received. Possible values: serial numbers of all AVT cameras connected to local machine are automatically loaded into the drop down menu.
  • Format - sets the number of bits in one pixel. Possible values: Y8 \ YUV411\ YUV422 \ YUV444 \ RGB8 \ Y16 \ RGB16 \ SY16 \ SRGB16 \ RAW8 \ RAW16. Note that not all formats are available, depending on the camera’s model.
  • FrameRate - sets the number of grabbed frames per second. Possible values: 3.75fps. \ 7.5 fps. \ 15 fps. \ 30 fps. \ 60 fps. This characteristic appears only if resolution is set to 640 x 480.
  • Resolution - sets the number of pixels in frame. Possible values: 160 x 120 \ 320 x 240 \ 640 x 480 \ 800 x 600 \ 1024 x 768 \ 10280 x 960 \ 1600 x 1200 \ Scalable. Note that FrameRate characteristic appears only if resolution is set to 640x480. Note that ROI characteristic appears only if resolution is set to scalable.
  • Brightness - sets the constant which should be added to the number of electrons in each pixel. Possible values: 0 - 255.
  • Gamma - use \ don’t use gamma correction (a method of non linear brightness correction aimed to clear up images). Possible values: 0 \ 1.
  • Shutter - sets exposure time in units of 20 microseconds. Possible values: 0 - 4095. Note that setting shutter value sets Shutt_μs values after reloading avt3_111 dialog box. Setting Shutt_μs, however, doesn’t affect Shutter value even after reloading the dialog box, though the shutter IS set to the value in Shutt_μs[1] [2].
  • Gain - sets brightness amplification. Possible values: 0 - 680. Note that brightness characteristic increases the number of electrons in each pixel by constant whereas gain characteristic increases it exponentially5.
  • ROI - sets the camera’s Region Of Interest (Xleft, Ytop, Xright, Ybottom). Possible values: Xleft: 0 - image width, Ytop: 0 - image height, Xright: Xleft - image width, Ybottom: Ytop - image height. All coordinates must divide by 4 without reminder (if they don’t, the program will automatically change them so they do). In order to return to full frame, set the ROI to (-1, -1, -1, -1). Generally, the smaller the ROI rectangle, the greater the frame rate. Note that ROI characteristic appears only if format is set to ‘Scalable’ and that it is relative to the full frame (for example, if ROI is set to (10, 10, 30, 30) and then set again to (10, 10, 30, 30), the same portion of the frame will be shown).
  • Trigger - use \ don’t use hardware trigger (grab frame when an appropriate signal is received in camera). Possible values:

    - off - don’t use hardware trigger. - Tr1 - use hardware trigger on the signal’s rise time and only if it was received in the 1st trigger input.
    - Tr1Inv - use hardware trigger on the signal’s fall time and only if it was received in the 1st trigger input.
    - Tr2 - use hardware trigger on the signal’s rise time and only if it was received in the 2nd trigger input.
    - Tr2Inv - use hardware trigger on the signal’s fall time and only if it was received in the 2nd trigger input.
    - Shutt_μS - sets exposure time in microseconds. Possible values: 1 - 80000. Note that setting Shutt_μs doesn’t affect Shutter value even after reloading the dialog box, though the shutter IS set to the value in Shutt_μs[3].

  • FrameInfo
  • TrigDelay_μS - sets delay between the time hardware trigger received and the time the frame was grabbed in microseconds. Possible values: 0 - 4,000,000,000.
  • Grab - sets the number of frames to grab. Note that Grab parameter is functional only if graph is running. Setting ‘Garb’ value to -1 sets it to the max possible value, 4,000,000,000.

[1] These restrictions originate from AVT driver.

[2] Checking the check box to the right of the value enables the automatic setting of the parameter by AVT driver.

[3] These restrictions originate from AVT driver.
