PtGrey1_8 gadget
 
   Input Formats:      transparent, used for software triggering

Outputs Formats:  CVideoFrame

Duplex Formats:    text

Location in gadget tree: Captures\Video\capture for grouped by type or Video\Capture by default

PtGrey1_8 gadget streams video captured by Point Grey Research camera with IEEE1394a interface through 1.* version of PGR SDK.

Video frames can be streamed via PG camera in the following methods:

  • No trigger - in which the camera grabs frames according to the predetermined frame rate set in setup dialog. In this method, Trigger must be set to ‘Off’.
  • Hardware trigger - in which external hardware (controller) is connected to the camera and triggers the grabbing of frames. In this method, Trigger must be set to ‘On’ and ‘TriggerPolarity’ must be set to the appropriate value (Plus \ Minus) to enable the receiving of the DoGrab signal.
  • Software trigger - in which the camera grabs one frame per data packet received in input pin. In this method, Trigger must be set to ‘Off’.

A complete description of current setup dialog parameters is printed to a text data packet and sent to control pin as a response to text data packet containing the command ‘list’ received in control pin. The value of a specific setup dialog parameter is written to a text data packet and sent to control pin as a response to another text data packet received in control pin containing one of the following commands:

  • get VideoFormat - invokes the sending of ‘VideoFormat’ value to control pin. Possible values: 5 \ 15 where 5 = 640x480mono, 15 = CustomVideoMode.
  • get FrameRate - invokes the sending of ‘FrameRate’ value to control pin. Possible values: 2 - 4, 6 where 2 = 7.5fps, 3 = 15fps, 4 = 30fps, 6 = 60fps.
  • get Brightness - invokes the sending of ‘Brightness’ value to control pin. Possible values: 0 - 255. • get Exposure - invokes the sending of ‘Exposure’ value to control pin. Possible values: 7 - 62.
  • get Gamma - invokes the sending of ‘Gamma’ value to control pin. Possible values: 0 \ 1.
  • get Pan - invokes the sending of ‘Pan’ value to control pin. Possible values: 0 - 112.
  • get Shutter - invokes the sending of ‘Shutter’ value to control pin. Possible values: 1 - 580.
  • get Gain - invokes the sending of ‘Gain’ value to control pin. Possible values: 16 - 64.
  • get Triggerdelay - invokes the sending of ‘Triggerdelay’ value to control pin. Possible values: 0 - 4095.
  • get ROI - invokes the sending of ‘ROI’ value to control pin. Possible values: Xleft: 0 - image width, Ytop: 0 - image height, Xright: Xleft - image width, Ybottom: Ytop - image height. Note that ROI parameter appears only if format is set to ‘CustomVideoMode’ and that it is relative to the full frame (for example, if ROI is set to (10, 10, 30, 30) and then set again to (10, 10, 30, 30), the same portion of the frame will be shown).
  • get Trigger - invokes the sending of ‘Trigger’ value to control pin. Possible values: 0 \ 1.
  • get TriggerPolarity - invokes the sending of ‘TriggerPolarity’ value to control pin. Possible values: 0 = Minus \ 1 = Plus. This parameter appears only if ‘Trigger’ is set to ‘On’.
  • get FrameInfo - invokes the sending of ‘FrameInfo’ value to control pin. Possible values: 0 = no additional information is sent with the frame / 1 = additional information is sent with the frame.

The value of a specific setup dialog parameter can be altered as a response to text data packet containing the command ‘set’, the parameter’s name and its value received in control pin, for example: set Shutter (480) will set the Shutter parameter to 480. Camera setup:

  • Camera - sets the PG camera from which the video stream is received. Possible values: camera type combined with serial numbers of all PG cameras connected to local machine are automatically loaded into the drop down menu.
  • VideoFormat - sets the number of bits in one pixel. Possible values: 640x480 mono \ CustomVideoMode. • FrameRate - sets the number of grabbed frames per second. Possible values: 7.5 fps. \ 15 fps. \ 30 fps. \ 60 fps. This parameter appears only if resolution is set to 640x480mono.
  • Brightness - sets the constant to be added to pixel intensity. Possible values: 1 - 255[1].
  • Exposure - sets the desired average pixel intensity of the frame. Functional only if Shutter or \ and Gain is \ are set automatically. Possible values: 7 - 62 [1].
  • Gamma - use \ don’t use gamma correction (a method of non linear brightness correction aimed to clear up images). Possible values: 0 \ 1.
  • Pan - sets the horizontal shift of the image. This parameter appears only if resolution is set to 640x480mono. Possible values: 0 - 112.
  • Shutter - sets exposure time in units of 125 microseconds. Possible values: 1 – 580[1].
  • Gain - sets brightness amplification. Possible values: 16 - 64. Note that brightness increases pixel intensity by constant whereas gain increases it exponentially[1].
  • Triggerdelay - sets delay between the time hardware trigger received and the time the frame was grabbed in microseconds. Possible values: 0 - 4095.
  • ROI - sets the camera’s Region Of Interest (Xleft, Ytop, Xright, Ybottom). Possible values: Xleft: 0 - image width, Ytop: 0 - image height, Xright: Xleft - image width, Ybottom: Ytop - image height. All coordinates must divide by 4 without reminder (if they don’t, Stream Handler automatically changes them so that they do). In order to return to full frame, set the ROI to (-1, -1, -1, -1). Generally, the smaller the ROI rectangle, the greater the frame rate. Note that ROI parameter appears only if format is set to ‘CustomVideoMode’ and that it is relative to the full frame (for example, if ROI is set to (10, 10, 30, 30) and then set again to (10, 10, 30, 30), the same portion of the frame will be shown).
  • Trigger - use \ don’t use hardware trigger (grab frame when an appropriate signal is received in camera). Possible values: on \ off.
  • TriggerPolarity - determines whether the grabbing of a frame will be done on the trigger signal’s rise time or drop time. This parameter appears only if Trigger is set to ‘On’. Possible value: Plus (rise time) \ Minus (drop time).
  • FrameInfo - determines whether additional information regarding the captured frame will be added to it (thus enabling, for example, the receiving of the frame’s ROI in ScriptRunner gadget which can, based on that information, set a different ROI for next frame and track an object). Possible values: On \ Off.

[1] Checking the check box to the right of the value enables the automatic setting of the parameter by PG driver
