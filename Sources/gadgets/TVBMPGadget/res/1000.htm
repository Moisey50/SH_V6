BMPCapture gadget
 
   Purpose: Streaming from image files.

Input Formats:     ANY

Outputs Formats: vframe

Duplex Formats:   NONE

Location in Gadget tree:  Captures\Files\Capture\BMPCapture (Grouped by type) or Files\Capture\BMPCapture (Grouped by description)

Data Processing – BMPCapture streams the bmp (or other formats files if installed gppflt.dll) files loaded to it. Files can be loaded by typing file location. If file location ends with *.bmp, all the bmp files in that location will be streamed. BMPCapture streams one frame per data packet received in input pin. Frame rate can be determined in setup dialog (possible values: 1 - 25) and is only relevant if no data packets are received in input pin. If loop drop box is set to true, the bmp files will be streamed in loop.

Setup:

Setup parameter FilePath describe path to source directory. The wildcards or full file name in the end of the request string is required.

Frame Rate describes frames rate in case of leak of connection to input synchronization pin.

LabelFullPath describes required or not to insert fullpath name of source file to a label of output datapacket.

The gadget has globally two modes WatchNew (false/true). In case of WatchNew == false gadget works in simple mode:

In this case we have 2 options: Loop file reading or not. In case of Loop==false - the gadget reads directory to the end and stops. otherwise it will read same directory in loop continuously.

The second option WatchNew==true is more complex:

in this case the Gadget reads all directory and stops, waiting new files in directory, if any appear, it will reads them and stops until new ones appear. You can set flag DeleteRead to true for case of requirements to delete all read files.
