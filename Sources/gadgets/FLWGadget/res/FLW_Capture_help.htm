  1. Setup Dialog:

FLWCapture Setup Dialog

  1. Data Processing – FLWCapture streams the flw file loaded to it. Files can be loaded via browse button or by typing file location. Flw files are video files that stream the frames in the same rate as they were captured and not in a predetermined or trigger sensitive rate (like avi files’ 25 frame \ second or each time data packet is received) unless the const rate check-box (left of the text ‘Frame rate’) or software trigger check-box are checked. If software trigger check-box is checked, FLWCapture streams one frame per data packet received in input pin. If const rate check-box is checked, FLWCapture streams the frames in the rate set in setup dialog (possible values: 1 to 25). If loop check-box is checked, the flw file will be streamed recursively. A complete description of current setup dialog parameters is printed to a text data packet and sent to control pin as a response to text data packet containing the command ‘list’ received in control pin.

The value of a specific setup dialog parameter can be printed to a text data packet and sent to control pin as a response to the following commands:

·        getmax - invokes the sending of the total number of frames in the FLW file to control pin.

·        getmaxtime - invokes the sending of the time value of the last frame in the FLW file to control pin.

·        get FileName - invokes the sending of ‘FileName’ parameter value to control pin.

·        get LoopFilm - invokes the sending of ‘LoopFilm’ parameter value to control pin. Possible values: true / false.

·        get SWTrigger - invokes the sending of ‘SWTrigger’ parameter value to control pin. Possible values: true / false.

·        get ConstRate - invokes the sending of ‘ConstRate’ parameter value to control pin. Possible values: true / false.

·        get FrameRate - invokes the sending of ‘FrameRate’ parameter value to control pin. Possible values: 0 - 25. 0 = FrameRate check box is not checked.

The value of setup dialog parameters FileName, LoopFilm, SWTrigger, ConstRate and FrameRate can be altered as a response to text data packet containing the command ‘set’, the parameter’s name and its value received in control pin, for example: set LoopFilm(true) will set the LoopFilm characteristic to true.

In addition, the command seek (FrameNumber) sets current frame to be FrameNumber and the command seektime(Time) sets current frame to be the frame with the time value equals to Time.
